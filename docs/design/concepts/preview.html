Service-illustration concept gallery

Phase 3 deliverable. Each card shows the outcome key-frame — the static state held after the process beats resolve, and the state shown in prefers-reduced-motion: reduce mode. Motion choreography is described per card. Two alternate directions per service are listed in the collapsed details if the strong concept doesn’t land in critique.

01 / Strong

Web Application Development

“Prototype to production-grade — without the months between.”

Process beats: shards drift in off-canvas → snap to invisible grid → thin lines draw between them. Outcome: consolidated into one device frame with a gold corner accent that lands and holds.

Alternate directions (not sketched)
  • B — Wireframe to fidelity: a wireframe rectangle grows in stroke weight in 3 steps until it becomes the polished frame. Read as “sketch → ship” rather than “parts → whole.”
  • C — Build dependency reveal: a single rect appears, then 4 smaller rects fade in connected by lines, suggesting a component tree settling. More technical, leans engineering-buyer.
02 / Strong

Cloud Infrastructure & DevOps

“One platform to operate, not a rack to maintain.”

Process beats: three server towers with blinking status dots → towers dissolve into ascending particles → particles drift to a convergence point. Outcome: coalesce into one cloud node with a single gold pulse-ring that lands and holds.

Play-once~2.4s arc + 480ms settle4 tween channels
Alternate directions (not sketched)
  • B — Pipeline collapse: a horizontal multi-stage pipeline (commit → build → test → deploy) collapses sideways into a single “deployed” node. Stronger DevOps narrative, weaker on the consolidation story.
  • C — Region replication: a node appears, replicates twice across the frame with thin connection lines, then the three nodes settle in a single triangle. Reads as “high availability” rather than “OpEx reduction.”
03 / Strong

Data Analytics & Dashboards

“Signal extracted from your noise — daily, without you asking.”

Process beats: dense scatter of ~30 dots → filter line sweeps left-to-right, dimming dots above → surviving dots reorganise into sequence. Outcome: ascending sparkline with a gold endpoint dot and quiet halo.

Play-once~2.6s arc + 480ms settle3 tween channels
Alternate directions (not sketched)
  • B — Histogram resolves: a noisy histogram normalises into a single distribution curve. More statistically literate, less universally readable.
  • C — Funnel compression: a wide funnel narrows to a single output bar. Stronger conversion-metric read, weaker on the “noise → signal” story.
04 / Strong

Digital Transformation Advisory

“Decision velocity — direction agreed before the next quarter starts.”

Process beats: tangled bezier knot pulses → one strand pulls taut horizontally → remaining strands untangle and fade. Outcome: single forward arrow with gold arrowhead. Faint ghost-trails preserved as memory of the prior tangle.

Play-once~2.5s arc + 480ms settle4 tween channels
Alternate directions (not sketched)
  • B — Decision tree prune: a branching tree of paths appears; non-chosen branches fade out, leaving one bold path. More literal, less elegant.
  • C — Compass settle: a multi-arrow compass spins, decelerates, and settles pointing forward. Risks reading as “direction-finding” rather than “decision-making.”
05 / Strong

API Development & Integration

“Three systems behaving as one — without rewriting any of them.”

Process beats: three horizontal channels pulse at different rhythms → sync marker descends through them → rhythms align. Outcome: channels merge into one thicker stream with a single gold marker travelling through.

Play-once~2.7s arc + 480ms settle4 tween channels
Alternate directions (not sketched)
  • B — Handshake nodes: two separate node graphs grow toward each other and join by a single connecting edge. More integration-pattern-literal, less “synchronised flow.”
  • C — Request/response pair: a request arrow goes left-to-right and a response arrow returns right-to-left in tight sequence. Cleanest for technical buyers, weakest for SMB.
06 / Strong · LOOP

AI Agent Engineering

“Operations that run themselves — and notify you only when they shouldn’t.”

Process beats: thin horizontal lines stack from the bottom (request queue) → vertical beam descends from top through the stack → lines collapse through the beam. Outcome: single answer-line exits at top with a quiet gold glow. Loops with a 1.5s pause between cycles — the only looping illustration.

LOOP · 1.5s gap~3.0s arc + 480ms settle4 tween channels
Alternate directions (not sketched)
  • B — Continuous filter: a stream of dots flows left-to-right; mid-stream, a vertical filter line classifies them into two outflow lanes. More agent-classifier-literal.
  • C — Sentinel pulse: a single node sits center; concentric rings emanate outward at irregular intervals (representing event-driven invocations). Most restrained option, weakest narrative.
