# .impeccable.md — Prudentia Digital

Design context for all impeccable / animate / shape / polish / critique / craft skill invocations on this project. Read this before any design work.

## Design Context

### Users

Three overlapping buyer personas, all reading the same surface:

1. **SA SMB operator / founder** evaluating an agency on a lunch break. Pragmatic. Comparing 2–3 alternatives. Needs proof of competence + ZAR price comfort + speed signals.
2. **Enterprise / mid-market IT or transformation lead** browsing between meetings. Risk-averse. Needs methodology reassurance, compliance signals, references.
3. **Technical buyer (CTO / engineering manager)** scrutinising how we work, not just outcomes. Skeptical of marketing speak. Wants craft, opinions, depth.

The site must read **credible across all three simultaneously**. SMB drives volume; enterprise drives upside; technical drives trust that converts both. No persona gets a separate page.

Primary job-to-be-done: "Decide in under 90 seconds whether this studio is worth a discovery call." Secondary: "Forward a link to a stakeholder without embarrassment."

### Brand Personality

Three words: **considered, quietly confident, restrained.**

Like a senior advisor speaking softly because they don't need to raise their voice. The brand name is Latin for *prudence* — that's not decorative, it's the operating instruction. Confidence comes from restraint, not from flourish.

Emotional target: **trust through stillness.** Not delight, not urgency, not delight-by-surprise. The viewer should think "these people know what they're doing" within 2 seconds and "I would let them touch my systems" within 30.

### Aesthetic Direction

**Visual frame:** the existing navy `#0D1B2A` + gold `#C9A96E` palette + DM Serif Display (headings) + Inter (body) is locked. This treatment already executes "considered" well. Don't fight it — animate within it.

**Reference points (positive):**
- *vite.dev hero* — for the PATTERN of "single declarative motion paired with explanatory copy in a 2-col grid." Borrowed: structure, scroll-paused playback, IO observer integration. NOT borrowed: lightning vocabulary, rainbow gradients, speed-as-message.
- *Klim Type Foundry's site* — slow, dense, unimpressed. The model for "motion is rare because it has to earn its place."
- *Stripe Press book pages* — restraint with one decisive accent per spread.

**Anti-references (explicit):**
- **Generic SaaS marketing site (Stripe / Linear / Vercel clones)** — glowing gradients, glassmorphic cards, perfectly-centered hero loops, snap-to-grid animations, cyan accents on dark. This is the "AI slop" default and the highest probability failure mode. If a designer can mistake the result for a Linear screenshot, we have failed.
- Big-4 consulting visual language (abstract triangles, stock-photo diversity panels, blue-and-white grids).
- African tech-startup template (neon greens/purples on dark, motion that screams for attention).
- Web3/crypto chrome (cyan-on-dark, perspective-warped 3D, hyper-glossy).

**Theme:** light-only currently (auto dark mode was reverted per recent commit). Illustrations must still theme-adapt via `currentColor` + CSS variables so dark mode can be re-enabled without rework.

### Framing Decision

**Hybrid — process implied, outcome emphasised.** Per-service animations open with 2–3 distinct beats showing *how* the work happens (process), then resolve into a single settled outcome state (the saving). The process beats are quick and quiet; the outcome beat lands and holds. Read as: "we work like this … so that you get this."

This is the higher-rigor option. It costs more design effort than pure outcome metaphors, but it's the only framing that flexes across all three personas simultaneously: SMB sees the outcome, enterprise sees the methodology, technical sees the craft.

### Design Principles

1. **Stillness over motion.** A frame of held stillness reads more confident than a frame of activity. The settle beat (~400–600ms of held outcome) carries more weight than the entry. Negative space is the dominant material, not the gold accent.

2. **Each animation is a sentence with a period.** Two or three process beats (commas), one outcome resolution (period). Then the loop pauses for ≥ 2 seconds before re-entering. No restless loops. No breath cycles. Pulse only on the AI Agent card (one exception, because the metaphor demands it).

3. **One gold moment per illustration.** Gold is reserved for the outcome accent — typically the final settled state's stroke, halo, or tip. Process beats are navy-on-warm-white. Restraint makes the gold land.

4. **Variety from shape language only.** Typography, colour, easing, settle timing, halo treatment — all shared across the six. The six services are visually distinguished by *what shape moves*, never by treatment. This prevents the page reading as a zoo.

5. **Pass the SaaS-clone smell test, always.** Before shipping any illustration: would a designer mistake it for a Linear or Stripe loop? If yes, rework. Specifically: no centred glowing orbs, no auto-snapping grid cards, no breathing gradients, no cyan accents anywhere on the page (not even by accident in mid-tween).

### Constraints (technical, carried forward)

- Vanilla HTML/CSS/JS only. Zero new runtime dependencies. Rive/Lottie/framer-motion all rejected.
- GPU-only animation properties: transform, opacity, filter. No width/height/margin/padding animations.
- `prefers-reduced-motion: reduce` → static key-frame SVG, no JS observer activity.
- Browser baseline: Chrome/Edge ≥ 111, Safari ≥ 16.4, Firefox ≥ 128.
- Shared motion vocabulary: `--anim-illustration-ease` (ease-out-quart), `--anim-illustration-settle` (400ms), `--anim-illustration-halo-blur` (24px). Lives in `css/design-tokens.css`.
- Typography is locked (DM Serif Display + Inter). Inter is on impeccable's reflex-font reject list; the monoculture risk is accepted because typography is out of scope for this work. Revisit on branding refresh.

### Out of Scope

- Copy rewrites quantifying savings (saved N hours / week, etc.).
- Typography changes.
- Touching `/ai/` page beyond the home-page AI Agent card illustration.
- Migration to a framework or build pipeline.

## Motion principle update (Phase 8 · 2026-05-17)

The earlier principle *"stillness over motion · each animation is a sentence with a period · play-once then hold"* is **suspended for the home-services grid only**. After user review of the Phase 7 result, the services-grid illustrations now use a **continuous 6s loop** (2s build → 2s hold → 1s fade-out → 1s breath) with **in-place entry** (no `scaleX` from edge, no `translateY` from above, no `stroke-dashoffset` draw — opacity fade and scale-from-center only).

The "stillness over motion" principle still governs the rest of the site (hero, why-us, portfolio, footer, etc.). It is *not* a global brand voice retraction — only a scoped override for the services grid where user feedback pushed against the original direction.

Future `/impeccable craft` or `animate` invocations:
- **Inside `.service-illustration*`**: use the loop pattern documented in `docs/design/service-animation-brief.md` ("Phase 8 override" section).
- **Elsewhere**: the original stillness/restraint principles still apply.
