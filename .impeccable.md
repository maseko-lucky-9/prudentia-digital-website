# .impeccable.md — Prudentia Digital

Design context for all impeccable / animate / shape / polish / critique / craft skill invocations on this project. Read this before any design work.

## Design Context

### Users

Three overlapping buyer personas, all reading the same surface:

1. **SA SMB operator / founder** evaluating an agency on a lunch break. Pragmatic. Comparing 2–3 alternatives. Needs proof of competence + ZAR price comfort + speed signals.
2. **Enterprise / mid-market IT or transformation lead** browsing between meetings. Risk-averse. Needs methodology reassurance, compliance signals, references.
3. **Technical buyer (CTO / engineering manager)** scrutinising how we work, not just outcomes. Skeptical of marketing speak. Wants craft, opinions, depth.

The site must read **credible across all three simultaneously**. SMB drives volume; enterprise drives upside; technical drives trust that converts both. No persona gets a separate page.

Primary job-to-be-done: "Decide in under 90 seconds whether this studio is worth a discovery call." Secondary: "Forward a link to a stakeholder without embarrassment."

### Brand Personality

Three words: **considered, quietly confident, restrained.**

Like a senior advisor speaking softly because they don't need to raise their voice. The brand name is Latin for *prudence* — that's not decorative, it's the operating instruction. Confidence comes from restraint, not from flourish.

Emotional target: **trust through stillness.** Not delight, not urgency, not delight-by-surprise. The viewer should think "these people know what they're doing" within 2 seconds and "I would let them touch my systems" within 30.

### Aesthetic Direction

**Visual frame:** the existing navy `#0D1B2A` + gold `#C9A96E` palette + DM Serif Display (headings) + Inter (body) is locked. This treatment already executes "considered" well. Don't fight it — animate within it.

**Reference points (positive):**
- *vite.dev hero* — for the PATTERN of "single declarative motion paired with explanatory copy in a 2-col grid." Borrowed: structure, scroll-paused playback, IO observer integration. NOT borrowed: lightning vocabulary, rainbow gradients, speed-as-message.
- *Klim Type Foundry's site* — slow, dense, unimpressed. The model for "motion is rare because it has to earn its place."
- *Stripe Press book pages* — restraint with one decisive accent per spread.

**Anti-references (explicit):**
- **Generic SaaS marketing site (Stripe / Linear / Vercel clones)** — glowing gradients, glassmorphic cards, perfectly-centered hero loops, snap-to-grid animations, cyan accents on dark. This is the "AI slop" default and the highest probability failure mode. If a designer can mistake the result for a Linear screenshot, we have failed.
- Big-4 consulting visual language (abstract triangles, stock-photo diversity panels, blue-and-white grids).
- African tech-startup template (neon greens/purples on dark, motion that screams for attention).
- Web3/crypto chrome (cyan-on-dark, perspective-warped 3D, hyper-glossy).

**Theme:** light-only, indefinitely. Auto dark mode was reverted earlier and is now **off the table** (decision: 2026-05-18). Stop preserving theme-adaptable scaffolding for a future re-enable — simplify when an opportunity arises. Existing `currentColor` + CSS variable patterns can stay where they don't cost effort, but no new code should pay a tax for dark mode.

### Framing Decision

**Hybrid — process implied, outcome emphasised.** Per-service animations open with 2–3 distinct beats showing *how* the work happens (process), then resolve into a single settled outcome state (the saving). The process beats are quick and quiet; the outcome beat lands and holds. Read as: "we work like this … so that you get this."

This is the higher-rigor option. It costs more design effort than pure outcome metaphors, but it's the only framing that flexes across all three personas simultaneously: SMB sees the outcome, enterprise sees the methodology, technical sees the craft.

### Design Principles

1. **Stillness over motion.** A frame of held stillness reads more confident than a frame of activity. The settle beat (~400–600ms of held outcome) carries more weight than the entry. Negative space is the dominant material, not the gold accent.

2. **Each animation is a sentence with a period.** Two or three process beats (commas), one outcome resolution (period). Then the loop pauses for ≥ 2 seconds before re-entering. No restless loops. No breath cycles. Pulse only on the AI Agent card (one exception, because the metaphor demands it).

3. **One gold moment per illustration.** Gold is reserved for the outcome accent — typically the final settled state's stroke, halo, or tip. Process beats are navy-on-warm-white. Restraint makes the gold land.

4. **Variety from shape language only.** Typography, colour, easing, settle timing, halo treatment — all shared across the six. The six services are visually distinguished by *what shape moves*, never by treatment. This prevents the page reading as a zoo.

5. **Pass the SaaS-clone smell test, always.** Before shipping any illustration: would a designer mistake it for a Linear or Stripe loop? If yes, rework. Specifically: no centred glowing orbs, no auto-snapping grid cards, no breathing gradients, no cyan accents anywhere on the page (not even by accident in mid-tween).

### Constraints (technical, carried forward)

- Vanilla HTML/CSS/JS only. Zero new runtime dependencies. Rive/Lottie/framer-motion all rejected.
- GPU-only animation properties: transform, opacity, filter. No width/height/margin/padding animations.
- `prefers-reduced-motion: reduce` → static key-frame SVG, no JS observer activity. **Also gate pointer-reactive layers** (tilt, glow, parallax) behind `@media (hover: hover) and (pointer: fine)` so touch devices skip them too.
- Browser baseline: Chrome/Edge ≥ 111, Safari ≥ 16.4, Firefox ≥ 128. Scroll-linked enhancements (`animation-timeline: view()`) ship as progressive enhancement under `@supports`; fallback is IntersectionObserver `.reveal`.
- Shared motion vocabulary: `--anim-illustration-ease` (ease-out-quart), `--anim-illustration-settle` (400ms), `--anim-illustration-halo-blur` (24px). Lives in `css/design-tokens.css`.
- Typography is locked (DM Serif Display + Inter). Inter is on impeccable's reflex-font reject list; the monoculture risk is accepted because typography is out of scope for this work. Revisit on branding refresh.

### Out of Scope

- Copy rewrites quantifying savings (saved N hours / week, etc.).
- Typography changes.
- Touching `/ai/` page beyond the home-page AI Agent card illustration.
- Migration to a framework or build pipeline.

## Motion principle update (Phase 8 · 2026-05-17)

The earlier principle *"stillness over motion · each animation is a sentence with a period · play-once then hold"* is **suspended for the home-services grid only**. After user review of the Phase 7 result, the services-grid illustrations now use a **continuous 6s loop** (2s build → 2s hold → 1s fade-out → 1s breath) with **in-place entry** (no `scaleX` from edge, no `translateY` from above, no `stroke-dashoffset` draw — opacity fade and scale-from-center only).

The "stillness over motion" principle still governs the rest of the site (hero, why-us, portfolio, footer, etc.). It is *not* a global brand voice retraction — only a scoped override for the services grid where user feedback pushed against the original direction.

Future `/impeccable craft` or `animate` invocations:
- **Inside `.service-illustration*`**: use the loop pattern documented in `docs/design/service-animation-brief.md` ("Phase 8 override" section).
- **Elsewhere**: the original stillness/restraint principles still apply.

## Reactive-depth override (Phase 9 · 2026-05-18 · `.why` section)

The "stillness over motion" principle is **further scoped** for any **value-proposition card grid** that previously rendered as flat text on a coloured section (currently: `.why__grid`, next: `.testimonials`, `.team`, `.about`). User feedback on 2026-05-18 was that the prior `.why` section read **dull** because the cards had no surface and the rotating `.card-pulse` gold accent had visibly sharp arc edges. Both root-causes were treated as design failures, not just polish gaps.

### What changed in the design vocabulary

1. **Cards must have a surface.** Text-on-section is not a card. A real card needs (a) a soft gradient background that lifts off the section, (b) a 1px tinted border, (c) a soft shadow stack (`1px + 8px@−12px` blur), (d) `var(--card-radius)`, (e) `var(--space-6)` interior padding. Without all five, motion can't rescue it.

2. **Hover earns more than a colour shift.** On pointer-fine devices, a value-prop card responds to the cursor with (a) a radial gold glow that tracks the pointer (`--why-mx/my` driven by a single delegated RAF-throttled handler), (b) ≤ 4° 3D `rotateX/Y` derived from the same pointer data, (c) icon Z-parallax via `transform: translateZ()` so tilt creates real depth. All gated behind `@media (hover: hover) and (pointer: fine)`.

3. **Per-card storytelling via `data-story`.** Each card carries a `data-story="layers|shield|globe|pulse|…"` attribute that names its hover gesture (e.g., compass spins, shield rocks, line stretches). The gesture only fires on `.in-view:hover` so it never collides with the entrance choreography. Variety from gesture, not from styling — consistent with the older "variety from shape language only" principle.

4. **Entrance is now a sentence with a comma-comma-period.** Halo bloom → icon `stroke-dashoffset` trace → title slide-up 8px → body fade, chained via CSS `animation-delay`, total ≤ 1.0s. Replaces the prior single-stage `.reveal` fade. Scroll-linked enhancement via `@supports (animation-timeline: view())` for browsers that have it.

5. **Soften shared decorative motion when it shows visible edges.** The `.card-pulse::before` rotating conic-gradient previously had hard arc stops (`transparent → gold → gold → transparent` at 0°/60°). It is now a 4-stop feathered ramp (`transparent → 0.18α @15° → full @30° → 0.18α @45° → transparent @60°`). Applies project-wide, including services illustrations — softer is always better on this brand. The user calibrates on edge geometry (caps, joins, gradient stops); never ship hard edges as decoration.

6. **All card icons get `stroke-linecap="round" stroke-linejoin="round"` and `.why__icon-wrap svg [stroke]` rules.** Default butt/miter is treated as a bug.

### Scope

- **Applies to**: `.why` (shipped), `.testimonials` (next), `.about`/`.team` (candidates), any future value-proposition card grid.
- **Does not apply to**: hero, services grid (Phase 8 loop override still governs), portfolio thumbnails, footer, content sections. Stillness still rules there.

### Anti-patterns to avoid when extending this

- **Do not stack `::after` glow on a `.card-pulse` node** without checking whether the rotating sweep also uses `::after`. The Why work intentionally dropped a planned counter-sweep to keep `::after` available for the pointer glow. One pseudo per concern.
- **Do not enable numbered badges (`01·02·03·04`) without explicit user sign-off**. They shipped as opt-in (`data-show-index` on the grid) but stay off by default — adding visible IA elements is a copy decision, not a motion one.
- **Do not skip the `@media (hover: hover) and (pointer: fine)` gate** on tilt/glow. Touch devices jitter on tap if it's there.
- **Do not animate `width/height/padding/margin`** — only `transform`, `opacity`, `filter`, `stroke-dashoffset`, and CSS custom properties driving conic-gradient angles.

## Reactive-depth override Phase 10 (2026-05-18 · `.why` escalation)

User re-issued the original "dull / sharp covers" complaint after Phase 9 went live. Diagnosis ruled in cache staleness (all `?v=` strings had been pinned at 1.2 long before Phase 9 shipped, plus an active service worker), and the user explicitly chose "escalate the whole thing" — pushing past the Phase 9 budget.

### Additions to the design vocabulary

1. **Kill orbital motion in this section. Replace with a persistent ambient halo.** The rotating `.card-pulse::before` conic-gradient is overridden on `.why__item.card-pulse` only — scoped via selector specificity so services / testimonials / team / AI page (19 cards across the site) keep their existing softened rotating sweep. Halo breathes via slow opacity + scale oscillation (8 s, ease-in-out alternate). No arc, no head, no tail — eliminates the "sharp covers" critique structurally.

2. **Cards breathe continuously, not only on hover.** Subtle `translateY` + `scale(1.005)` over 7 s, ease-in-out alternate. Desynced per card via inline `--card-phase: 0s | -1.75s | -3.5s | -5.25s` so the four don't pulse in unison (uniform breath would read as a single zoom, not "alive"). Hover pauses the breath via `animation-play-state: paused` and the pointer tilt takes over.

3. **Connecting tendrils between cards (system-not-tiles signal).** `<svg class="why__tendrils">` overlay inside `.why__grid`. Three bezier paths (cards 1↔2, 2↔3, 3↔4) computed in JS from icon-wrap centre positions, drape ~14 px below the straight segment, opacity 0 by default. Hover any card → `data-tendril-active` on the grid lights the two incident tendrils. Only shown on `min-width: 1024px` + hover-fine (stacked layouts hide them — vertical lines don't carry the "system" metaphor).

4. **Bespoke metaphor icons replaced generic strokes.** Senior-led delivery → fountain pen nib + ink dot. 100% Black-owned → wax-seal disc with radial dashes. Regulatory fluency → folded document with corner fold + seal mark. One team, full stack → four small dots converging into a filled centre. Each ~24×24 stroke-based, round caps/joins inherited from the existing `.why__icon-wrap svg [stroke]` rule. The `.why__icon-path` draw-on trace from Phase 9 applies unchanged.

5. **Soft scroll-snap on stacked layouts only.** `scroll-snap-type: y proximity` on `.why` + `scroll-snap-align: start` + `scroll-snap-stop: normal` on `.why__item`, gated by `@media (max-width: 1023px)`. Gentle magnetize per card on mobile/tablet, no momentum-scroll trap. Above 1024 px all four cards are visible simultaneously so no snap is needed.

6. **Bigger lift on hover (6° tilt, was 4°).** Resting shadow is also deeper (`0 16px 36px -14px @0.16`). Icon Z-parallax pushed `translateZ(28px) → translateZ(36px)`. The card is now more obviously elevated even at rest, more dramatically lifted on hover.

7. **Cache-bust as part of the design change.** Every `?v=` string in `index.html` bumped (`1.2 → 1.3` for the v1.2 cohort; `?v=1.0 → 1.1-phase10` for `why-interactions.js`; `?v=0.2-loop → 0.2-loop-phase10` for service-illustrations.css even though that file didn't change). Service workers that key on URL treat these as new fetches.

### Scope

- **Applies to**: `.why` (shipped Phase 10), `.testimonials.populated-mode` (extension candidate — already has Phase 9 surface), `.team` (candidate).
- **Does NOT apply to**: hero, services grid, portfolio, footer, content sections. Phase 8 + original stillness/restraint still govern there.
- **Cross-section impact**: `.card-pulse::before` orbital sweep stays untouched on the 19 cards across services/testimonials/team/AI page — only `.why__item.card-pulse::before` is overridden via specificity.

### New anti-patterns

- **Do not propagate the persistent halo to other `.card-pulse` users without a separate review.** It's a section-character call: services are a *catalog* (rotating sweep reads as activity), `.why` is a *manifesto* (halo reads as authority). Different motion budgets.
- **Do not light all four tendrils on hover.** Only the two incident ones. Lighting all three reads as a network diagram, not as the card-to-neighbour gesture.
- **Do not animate the tendril `d` attribute on hover.** Path geometry is recomputed only on layout change (`ResizeObserver`). Hover changes `opacity` only — that's the cheap GPU op.
- **Do not let the continuous breath fire on touch.** Currently it does (it's *not* hover-gated) — only the tilt + glow + tendrils are. If mobile perf budget becomes a problem, gate the breath under `@media (hover: hover) and (pointer: fine)` too. For now the breath is single-element transform-only and free.
